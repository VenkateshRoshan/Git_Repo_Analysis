# Engineering Impact Dashboard

A dashboard that analyzes the GitHub repository and identifies the most impactful engineers over the last N days.

**Live dashboard :** [https://venkateshroshan.github.io/Git_Repo_Analysis/]

---

## What does "impact" mean here?

Counting commits or lines of code doesn't tell the full story. This dashboard measures impact across 3 dimensions:

- **Shipping (40%)** - Are they consistently merging PRs? This is the core job.
- **Collaboration (35%)** - Are they reviewing others' work and unblocking teammates?
- **Influence (25%)** - Is their work generating meaningful discussion? Do others engage with what they build?

```
Total Score = shipping_score x 0.4 + collaboration_score x 0.35 + influence_score x 0.25
```

Each dimension is normalized to a 0-100 scale relative to all contributors in the period, then combined into a single impact score. Normalization ensures no single metric dominates just because its raw numbers happen to be larger.

---

## How it works

```
python analysis.py  →  data.json  →  index.html reads it  →  hosted on GitHub Pages
```

1. `analysis.py` fetches all merged PRs from the last 90 days via the GitHub API
2. For each PR, it records author stats and fetches reviewers asynchronously
3. Scores are normalized and weighted into a final impact score
4. Everything is written to `data.json`
5. `index.html` reads the JSON and renders the dashboard - no backend needed

## How to Run

**Prerequisites:**
- Python 3.8+
- A GitHub personal access token (needs `public_repo` read scope)

**Setup:**
```bash
pip install -r requirements.txt
```

Create a `.env` file:
```
GITHUB_ACCESS_TOKEN=your_token_here
```

**Run:**
```bash
python analysis.py
```

Optional flags:
```bash
python analysis.py --days 60 --top-n 10 --shipping-weight 0.5 --collab-weight 0.3 --influence-weight 0.2
```

| Flag | Default | Description |
|------|---------|-------------|
| `--days` | 90 | How many past days to analyze |
| `--repo` | PostHog/posthog | GitHub repo to analyze |
| `--top-n` | 5 | Number of top contributors to show |
| `--shipping-weight` | 0.4 | Weight for shipping score |
| `--collab-weight` | 0.35 | Weight for collaboration score |
| `--influence-weight` | 0.25 | Weight for influence score |

> Note: The three weights must sum to 1.0.

## Known Limitations

- **Collaboration score may show 0** for all contributors if the GitHub API rate limit (5,000 requests/hour) is exhausted before review fetches complete. Fetching real `additions`/`deletions` for ~5,000 PRs uses one API call per PR, leaving insufficient quota for the reviews endpoint in the same run.
